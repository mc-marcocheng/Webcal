N個數的最小公倍數

程式編日期: 2006年9月2日

程式可以計算兩數的最大公因數。第一個程式較簡短，但程式需要在BASE模式執行，操作較不便(完結後需後按MODE 1返回正常計算模式)，第二個程式較長但在COMP模式執行，容許輸入的數值亦較大。

第一個程式(59 bytes，使用記憶為A、B、C、D及X)

程式需要在 BASE 模式下執行，因此在輸入程式前請先按 Mode Mode 3 進入BASE模式。

Dec: ?→A: Lbl 0: A→B: ?→A: A→D: B→X: Lbl 1:

B - B ÷ A × A→C => A→B => C→A => Goto 1: X÷A×D→A: Goto 0

　

第二個程式 (COMP模式，62 bytes，使用記憶為A、B、C及D)

?→A: Lbl 0: A→B: ?→A: AB→D: Lbl 1: B ÷ A - . 5: Fix 0: Rnd:

Norm 1: B - Ans A→C => A→B => C→A => Goto 1: D÷A→A: Goto 0

　

註: 輸入的數值必須為正整數，否則答案錯誤。

　

例題1: 找48及60的最大公因數。

按 Prog 1 再按 48 EXE 60 EXE (顯示最小公倍數為240)

計算完結後按AC 終止程式。

　

例題2: 找48、60及42的最大公因數。

按 Prog 1 再按 48 EXE 60 EXE 42 (顯示最小公倍數為1680)

計算完結後按AC 終止程式。

　

若果是第一個程式，計算完結後可直接按 MODE 1終止及返回正常計算模式
