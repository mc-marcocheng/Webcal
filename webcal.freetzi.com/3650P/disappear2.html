舊型號CASIO fx-3650P及3950P

應付程式自動消失方法

編寫日期: 2006年7月5日 (更新日期: 2010年1月1日)

當程式出現自動消失時，要花時間重新輸入消失的程式，實在令人覺得麻煩，若果在考試的時候出現，更加可能束手無策，只有不用程式就算了！那麼有沒有方法解決或減低消失的風險呢？經過很多不同情況的測試，以下是一些建議，嘗試克服Casio fx-3650P及fx-3950P這方面的問題。

　

1. 減少使用REPLAY功能:

就已知消失的方法，都與使用REPLAY功能有關，因此只要減少不必要使用REPLAY功能，自然可以減少程式自動消失的機會率。

　

2. 計數機自我檢測:

說明書ch-46指出當按 ON 時，計數機器會執行自檢操作，並在發現異常時將儲存在記憶器中的資料全部清除會清除。因此應盡量減少不必要按ON的操作，例如: 離開程式編輯模式按 MODE MODE MODE 2 代替直接 ON，一般計算使用 AC 代替 ON 等等。

　

3. 程式或計算異常中斷:

測試結果顯示，程式自動消失與程式的異常中斷有關，異常中斷包括以下三種情況: 第一種情況是出現任何 ERROR ; 第二種情況是程式(或非程式一般計算)在 計算機內部運算中，用家按 ON 或 AC 強行中斷 ; 最後一種情況是程式在要求輸入變數時，用家按 ON 或 AC 中斷程式。

當遇上以上的三種情況要特別小心，建議在出現上述三種情況時，立即轉換操作模式(可以轉換到相同模式)，例如: 按 MODE 1，這個方法經測試証明有效阻止問題出現，其實亦可以按 SHIFTMODE 2 EXE (亦即是 Mode clear) ，不過計數機的一些設定會改變。

　

4. 執行算式時出現黑色三角形◢:

當執行算式時有黑色三角形出現在算式未端(第一行)，這是一個很危險的訊號，所有已知消失的方法都會產生這個三角形(注意黑色三角形可以因執行的算式過長而未能顯示)，若果發現這個三角形，千萬不要按 REPLAY 向上的鍵，因為有問題的資料可能已經儲存到 REPLAY記憶中，這時應該立即清除REPLAY記憶以防止問題的出現，立即轉換操作模式(包括轉換到相同模式)，例如: 按 MODE 1，其實這時亦可以按 ON 消除 REPLAY 記憶，不過由於按ON時，計數機會自我檢測，當發現有異常時會清除所有程式，所以不建議這個方法。

　

5. 使用REPLAY時出現異常:

當嘗試用 REPLAY 上下找尋記錄時，出現異常的記錄(第一行或第二行的資料)，例如出現亂碼，有時異常會是在第一行中沒有任字符且沒有浮標，一旦出現這種異常情況，千萬不可以嘗試執行這個算式，亦千萬不可以按 ON，因為已可能觸及程式自動消失的因子。若果是觸及這個因子，這時即使執行其它算式並且不按 ON，這個消失的因子亦會一直保留在計數機中，直至下一次按 ON 時程式才會消失，這時要消除這個因子的唯一方法是按 SHIFT MODE 2 EXE (亦即是 MODE clear)，但計數機的一些設定會因此而改變。

　

6. 總結:

(1) 當程式執行中斷或者出現黑色三角形時，請立即轉換操作模式 (包括轉換至相同的模式)，例如按 MODE 1。

(2) 當使用 REPLAY 上下找尋記錄時，出現任何異常(包括第一行沒有任何字符而且沒有浮標)，請立即按 SHIFT MODE 2 EXE (MODE clear)。

　

7. 其它方法:

其實若果程式不需要配合計算機特別的內置功能一併使用，例如: 積分、複數功能等等，可以考慮將程式全部都設定在統計模式(SD或REG)，在統計模式之下的程式，除了有額外的變數記憶可以使用外，由於會自動取消REPLAY的功能，因此在統計模式之下，基本上是可以避免程式自動消失問題的發生，所以統計模式的程式可以說是很安全的模式。
