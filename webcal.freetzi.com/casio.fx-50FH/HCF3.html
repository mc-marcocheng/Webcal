N個數的最大公因數

程式編日期: 2014年8月22日

程式可以計算兩數的最大公因數。第一個程式較簡短，但程式需要在BASE模式執行，操作較不便(完結後需後按MODE 1返回正常計算模式)，第二個程式較長但在COMP模式執行，容許輸入的數值亦較大。

第一個程式(44 bytes，使用記憶為A、B及C)

程式需要在 BASE 模式下執行，因此在選擇新程式位置後，按 3 選用BASE模式。

Dec: ?→A: Lbl 0: A→B: ?→A: While Ans:

B - B ÷ A×A→C: A→B: C => C→A: WhileEnd: Goto 0

註: 如果是使用fx-50FH，上述程式中的乘號 ×可以省略不輸入，程式長度可節省1 byte。

　

第二個程式 (COMP模式，51 bytes，使用記憶為A、B及C)

?→A: Lbl 0: A→B: ?→A: Fix 0: While Ans:

B - A Rnd( B ÷ A - . 5→C: A→B: C => C→A:

WhileEnd: Norm1: Goto 0

　

註: 輸入的數值必須為正整數，否則答案錯誤。

　

例題1: 找48及60的最大公因數。

按 Prog 1 再按 48 EXE 60 EXE (顯示最大公因數為12)

計算完結後按AC 終止程式。

　

例題2: 找48、60及42的最大公因數。

按 Prog 1 再按 48 EXE 60 EXE 42 (顯示最大公因數為6)

計算完結後按AC 終止程式。

　

註: 若果是第一個程式，計算完結後可直接按 MODE 1終止及返回正常計算模式
